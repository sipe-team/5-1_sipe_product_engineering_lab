# 👋 참여자 소개

이 폴더에 본인의 자기소개와 미션 기대사항을 PR로 올려주세요.

## 작성 방법

1. 이 레포를 **Clone** 합니다
2. `members/{본인-github-username}` 브랜치를 생성합니다
3. `members/{본인-github-username}.md` 파일을 생성하고 아래 템플릿을 복사하여 작성합니다
4. Push 후 `main` 브랜치로 PR을 생성합니다

## 📝 템플릿

아래 내용을 복사해서 본인 파일에 붙여넣고 작성해주세요.

---

```md
# 👋 {이름}

## 기본 정보
- **직군**: (예: FE / BE / iOS / Full-stack / AI / DevOps 등)
- **경력**: (예: 3년차)

## 이 미션에 신청한 이유
> 단순 호기심이든, 이직 대비든, 실무에서 뭔가 부족함을 느꼈든 솔직하게 적어주세요!
> 예: "코드는 잘 짜는데 뭘 만들어야 할지 판단하는 능력이 부족하다고 느꼈어요"
> 예: "PM이 왜 그런 결정을 하는지 이해하고 싶어요"
> 예: "이직 면접에서 프로덕트 관점 경험을 어필하고 싶어요"



## 분석해보고 싶은 서비스나 도메인
> 본인 회사 제품이어도 좋고, 평소 관심 있던 서비스여도 좋아요.
> 여러 개 적어도 괜찮습니다!
> 예: "토스 — 송금 플로우가 왜 이렇게 설계되었는지 궁금해요"
> 예: "우리 회사 B2B SaaS 제품 — 온보딩 퍼널을 분석해보고 싶어요"



## 6주 뒤에 뭘 얻어가고 싶어요?
> 포트폴리오, 사고 프레임, 블로그 글, 실무 적용 등 뭐든 좋습니다.



## 선호하는 진행 방식 (선택)
> 어떤 방식이 본인에게 가장 도움이 될 것 같은지 편하게 적어주세요.
> 정해진 답은 없고, 이 답변을 바탕으로 OT에서 함께 구조를 정할 예정입니다.
>
> 예시:
> - 소그룹(3~4명) 토론 vs 전체(11명) 발표
> - 매주 산출물 vs 격주 발표
> - 온라인 vs 오프라인
> - 비동기 리뷰(GitHub 코멘트) vs 실시간 토론
> - 같은 서비스를 함께 분석 vs 각자 다른 서비스 분석


```

## 예시

> 아래는 작성 예시입니다. 참고만 해주세요!

### members/treejy.md

```md
# 👋 조태진

## 기본 정보
- **직군**: FE
- **경력**: 4년차
- **현재 회사/팀**: OO 스타트업 / 프로덕트팀

## 이 미션에 신청한 이유
실무에서 Jira 티켓을 받아서 치우는 패턴이 반복되다 보니,
"이 기능이 왜 필요한지"를 묻는 습관이 약해진 것 같아서 신청했습니다.
시니어로 성장하면서 기술 너머의 시야를 갖추고 싶어요.

## 분석해보고 싶은 서비스나 도메인
- 토스 — 간편 송금 이후의 금융 서비스 확장 전략
- 당근 — 지역 기반 서비스에서 커머스로의 확장

## 6주 뒤에 뭘 얻어가고 싶어요?
Why → What → KPI → How 가 실무에서 자연스럽게 나오는 습관을 만들고 싶어요.
분석한 내용을 기술 블로그로 정리해서 발행하는 것도 목표입니다.

## 선호하는 진행 방식
- 소그룹(3~4명)으로 나눠서 매주 깊게 토론하는 게 좋을 것 같아요
- 비동기 리뷰 + 모임에서 하이라이트 토론 조합이면 좋겠습니다
- 같은 도메인에 관심 있는 사람끼리 그룹이 되면 시너지가 날 것 같아요
```
