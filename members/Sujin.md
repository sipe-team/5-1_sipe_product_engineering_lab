# 김수진

## 기본 정보

- **직군**: 콘텐츠 도메인 프론트엔드 개발
- **경력**: 1년차 3개월 (인턴 1년 포함)
- **현재 회사/팀**: 포스타입 / 프론트엔드 팀. 컨슈머 파트 (조직 개편될 예정입니다.)

## 이 미션에 신청한 이유

- 회사에서 데이터 기반으로 유저 경험을 분석해서 개선하는 작업을 해보고 싶습니다.
- 불편함을 느낀 부분을 개선하고 나서 실제로 얼마나 개선했는지 정작 데이터 분석을 못해본 티켓들이 많아 이번 기회에 데이터 분석을 해보고 싶습니다.
- 신규 기능 개발 후에도 유저 반응이 어떤지 분석하지 못한 경우가 있다는 것도 발견했습니다. (ex. 명절 전 배포가 의미있는가 ?)
- 회사 서비스에 대해 잘 알고 있다며 생각하며 입사했으나, 여전히 구매자와 창작자간에 중요하게 생각해야되는 부분에 대해 잘 이해하지 못하고 있다는 생각이 들었습니다.
- 이번 미션을 통해 유저분들에게 더 나은 경험을 제공해드리며 서비스에 대해 더 깊이 이해해보고 싶습니다.

## 분석해보고 싶은 서비스나 도메인

## 6주 뒤에 뭘 얻어가고 싶어요?

실제로 오탈률, 에러 발생률이 높은 페이지를 찾아서 개선하고 얼마나 개선되었는지 데이터 분석 결과까지 보여드리고 싶습니다.

ex. app에서는 없지만 web에서만 유독 모바일로 접속 시 댓글 모달 뒤로가기 오탈률이 높아서 해당 문제를 해결.

## 선호하는 진행 방식 (선택)

비슷한 도메인 또는 B2C 서비스와 그룹을 지어서 진행하면 좋을 것 같습니다 !
