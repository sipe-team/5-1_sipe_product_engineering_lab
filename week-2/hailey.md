# 2주차: 사용자 플로우 역추적

## 서비스 정보

- **서비스:** Unified Alerting & Notification Platform (사내 Observability 플랫폼)
- **분석 기능:** 온보딩 → 첫 알람 설정
- **사용자 정의:** 새로 클러스터를 받은 게임 서비스 개발팀

## 한 줄 정리

> **이 기능은** 각 서비스 팀이 개별적으로 구축해야 했던 모니터링/알람 체계의 셋업 비용을 제거하는 **문제를 풀기 위해 존재하고,**
> **AARRR 중** Acquisition(클러스터 할당 시 자동 연동) → **Activation(첫 알람 수신까지의 아하 모먼트)** **단계를 움직이려는 것 같다.**

-----

## 추가 메모

- **아하 모먼트 후보:** 4단계(대시보드에서 데이터 확인)와 6단계(첫 알람 수신) 두 곳에 존재. 진짜 아하 모먼트는 6단계 — “내가 안 보고 있어도 알려준다”는 가치를 체감하는 순간
- **경영진 설득 포인트:** 이 온보딩 플로우가 없었다면 각 팀이 평균 N시간을 모니터링 셋업에 썼을 것 → 129개 서비스 × N시간 = 절감된 엔지니어링 시간으로 환산 가능
- **개선 아이디어:** 온보딩 완료율 추적 (클러스터 할당 → 첫 알람 설정까지 도달률)을 KPI로 잡으면 플랫폼의 Activation rate를 측정할 수 있음

### Datadog 대비 온보딩 비용 비교

|항목             |Datadog (각 팀이 직접)                                                                  |우리 플랫폼                                          |
|---------------|-----------------------------------------------------------------------------------|------------------------------------------------|
|Agent 설치       |Helm repo 추가 → Operator 설치 → API 키 Secret 생성 → DatadogAgent YAML 작성 → kubectl apply|클러스터 할당 시 사이드카 자동 설치 (개발자 작업 0)                 |
|수집 설정          |values.yaml에 로그/APM/메트릭 활성화 옵션 직접 작성                                               |pom.xml 디펜던시 추가 + Helm 차트 설정값 추가                |
|알람 설정          |Datadog UI에서 Monitor 생성 (별도 학습 필요)                                                 |자체 어드민 UI에서 규칙 생성 (사내 맥락에 최적화)                  |
|대시보드           |직접 구성 (패널 추가, 쿼리 작성, 변수 설정 등)                                                      |TPS·DB 상태·메트릭·로그 등 템플릿 기본 제공. 변수에서 프로젝트만 선택하면 완료|
|알림 채널          |Slack/이메일 등 외부 연동 설정                                                               |워크스 개인방/단체방 선택 (사내 메신저 즉시 연동)                   |
|**팀당 예상 셋업 시간**|**수 시간 (공식 문서 탐색 + 트러블슈팅 포함)**                                                     |**수십 분 이내**                                     |
|**129개 서비스 기준**|**수백 인시(man-hour) 소요**                                                             |**대부분 자동화로 절감**                                 |

→ Datadog 같은 업계 표준 SaaS를 쓰더라도 팀마다 초기 셋업 비용이 발생하는데, 우리 플랫폼은 사이드카 자동 설치 + 최소 설정 구조로 이 비용을 거의 0에 가깝게 만들었다. 이것이 “돈이 안 되는 플랫폼”이 아니라 “보이지 않는 비용을 절감하는 플랫폼”이라는 경영진 설득의 핵심 근거.
