# 2주차 - 사용자 플로우 역추적

## 분석 기준 (AARRR / User Journey)
- 이번 문서는 AARRR을 "단계명"이 아니라 "행동 결과 지표"로 해석했다.
- Acquisition: 기능 페이지 첫 유입이 발생했는가
- Activation: 사용자가 핵심 가치를 실제로 경험했는가
- Retention: 반복 방문/반복 사용 루프가 만들어졌는가
- Revenue: 거래/유료 전환 등 비즈니스 가치로 연결됐는가
- Referral: 공유/추천을 통해 신규 유입을 만들었는가

## 주제 A: 야구 앱 `세이버메트릭스 인사이트`

### 1) 기능 선택 이유
- 야구 팬은 전통 지표(타율, 평균자책)만으로는 설명되지 않는 경기 맥락을 이해하고 싶다.
- 세이버메트릭스는 어렵다는 진입장벽이 있어, "쉽게 해석해 주는 화면" 자체가 차별점이 된다.
- 경기 시청 전/중/후 모두 활용 가능해서 반복 학습과 재방문 루프를 만들기 좋다.

### 2) User Journey Map
| 단계 | 사용자 행동 | 감정/동기 | 서비스 설계 의도 | AARRR |
|---|---|---|---|---|
| 1. 진입 트리거 | 경기/선수 페이지에서 "고급지표 보기" 진입 | "요즘 이 선수 폼이 진짜 좋은 건가?" | 관심 선수 맥락에서 자연스럽게 고급지표 노출 | Acquisition, Activation |
| 2. 핵심 이해 | OPS, wRC+, FIP, BABIP 등 핵심 지표와 쉬운 설명 확인 | "숫자는 봤는데 의미를 모르겠음" | 지표 정의 + 리그 평균 비교를 함께 제공해 이해 부담 감소 | Activation |
| 3. 맥락 확장 | 기간/상대/구장 필터로 지표 변화 탐색 | "최근 한 달만 보면 어떤가?" | 탐색형 인터랙션으로 체류 시간 및 학습 깊이 증가 | Retention |
| 4. 인사이트 소비 | "오늘의 해석" 카드/요약 리포트 확인 | "한 줄로 정리해주면 좋겠다" | 복잡한 수치를 스토리로 번역해 재방문 동기 강화 | Retention |
| 5. 확산/다음 행동 | 카드 공유, 관심 선수 팔로우, 다음 경기 알림 설정 | "이 인사이트는 공유할 만함" | 공유와 팔로우를 통해 팬 커뮤니티 확산 유도 | Referral, Retention |

### 3) 역산 분석: 이 기능의 존재 이유
- 가설 1: 세이버메트릭스 인사이트는 "야구를 더 잘 이해하고 싶은 팬"을 장기적으로 락인시키는 학습형 기능이다.
- 가설 2: 복잡한 지표를 해석 가능한 정보로 변환하면 초중급 팬의 진입장벽을 낮춰 활성화율을 높일 수 있다.
- 가설 3: 경기 단위 소비를 넘어 선수/팀 단위 팔로우 루프를 만들어 비시즌에도 리텐션을 유지한다.

### 4) 핵심 지표 연결
- 핵심 KPI 후보: 고급지표 페이지 MAU, 지표 해석 카드 소비율, 선수 팔로우 전환율
- 직접 지표(Leading): 고급지표 진입률, 지표 설명 펼침률, 기간/필터 사용률
- 간접 지표(Lagging): 7일/30일 리텐션, 인사이트 공유율, 알림 설정률

### 5) 한 줄 결론
- 이 기능은 "야구를 더 깊게 이해하고 싶은 욕구"를 기반으로 학습형 리텐션과 팬 락인을 만들기 위해 존재한다.

## 주제 B: 네이버 금융 `리서치(증권사 리포트 탐색) 기능`

### 1) 기능 선택 이유
- 투자자는 매수/매도 전에 "신뢰할 만한 근거"를 빠르게 확인하고 싶다.
- 리포트 탐색은 뉴스 소비보다 의도가 높은 행동이라, 서비스의 정보 신뢰도를 평가하기 좋다.
- 실제 사용 흐름(종목 검색 -> 리포트 확인 -> 상세 읽기 -> 저장/공유)을 AARRR에 매핑하기 용이하다.

### 2) User Journey Map
| 단계 | 사용자 행동 | 감정/동기 | 불편했던 점 | 서비스 설계 의도 | AARRR |
|---|---|---|---|---|---|
| 1. 문제 인식 | 포털 검색/종목 페이지에서 리포트 탭 진입 | "이 종목을 숫자로 검증하고 싶다" | 원하는 종목 리포트를 찾기까지 탭 이동이 잦음 | 검색 유입을 종목 분석 행위로 연결 | Acquisition |
| 2. 필터링 | 최신순, 증권사, 산업 키워드로 리포트 추리기 | "지금 시점에 유효한 의견만 보고 싶다" | 제목만으로 품질 판단이 어려움 | 탐색 비용을 줄여 첫 읽기 행동 유도 | Activation |
| 3. 상세 읽기 | 목표주가, 실적 추정, 핵심 논리 확인 | "왜 이렇게 판단했는지 근거가 필요" | PDF/원문이 길어 핵심 파악에 시간이 걸림 | 체류를 늘리고 신뢰 경험 제공 | Activation, Retention |
| 4. 비교 검토 | 서로 다른 증권사 의견을 비교 | "의견이 갈리는 이유를 알고 싶다" | 리포트 간 용어/전제 차이로 비교 난이도 높음 | 반복 조회를 유도해 분석 루틴 형성 | Retention |
| 5. 후속 행동 | 관심종목 저장, 알림 설정, 증권 앱 이동 | "근거를 보고 실행 타이밍을 잡자" | 리포트 -> 실제 주문 전환은 앱 간 이동이 필요 | 정보 소비를 거래/재방문으로 연결 | Revenue, Retention |

### 3) 역산 분석: 이 기능의 존재 이유
- 가설 1: 리포트 기능은 "정보 신뢰 허브" 역할을 통해 투자 전환 직전의 핵심 니즈를 붙잡는다.
- 가설 2: 최신 리포트가 지속적으로 올라오는 구조로 사용자 재방문 루프를 만든다.
- 가설 3: 포털 내 정보 탐색 시간을 늘리고, 최종적으로는 제휴/연계 증권 서비스의 거래 행동에 간접 기여한다.

### 4) 핵심 지표 연결
- 핵심 KPI 후보: 리포트 탭 진입률, 리포트 상세 열람률, 사용자당 리포트 조회 수
- 직접 지표(Leading): 검색 -> 리포트 클릭 전환율, 리포트 완독률(또는 체류시간), 비교 조회율
- 간접 지표(Lagging): 관심종목 등록률, 증권 서비스 이동률, 7일/30일 리텐션

### 5) 한 줄 결론
- 이 기능은 "투자 판단의 근거를 빠르게 확보"하게 만들어 리텐션과 거래 전환 가능성을 함께 높이기 위해 존재한다.

## 최종 제출 방향
- 2개 주제 모두 작성해 두고, 리뷰 시에는 투자 주제를 메인으로 토론하는 방식이 적합하다.
- 이유: 투자 리포트 탐색 기능은 "원하는 정보를 가져간다"는 너의 관심사와 정확히 맞닿아 있고, AARRR 근거도 비교적 선명하다.
