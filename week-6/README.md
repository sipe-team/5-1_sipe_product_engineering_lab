# 📅 6주차: 최종 발표 — 프로덕트 분석 포트폴리오

## 이번 주 목표
- 5주간의 분석을 하나의 케이스 스터디 문서로 정리
- 최종 발표
- 전체 회고

## 최종 포트폴리오 구성

1. **서비스 소개** — 왜 이 서비스를 골랐는가
2. **사용자 플로우 분석** — 핵심 기능의 존재 이유 (2주차)
3. **공개 데이터 분석** — 성공/실패 추론과 근거 (3주차)
4. **개선 PRD** — 문제 → 솔루션 → 우선순위 (4주차)
5. **경쟁 분석** — 도메인 인사이트 (5주차)
6. **배운 점** — 사고방식 변화, 실무 적용 계획

## 발표 가이드
- 팀/개인당 15~20분 + Q&A
- 발표 자료 형식 자유 (슬라이드, 문서, 노션 등)

## 회고 프레임 (KPT)
- **Keep**: 이 미션에서 계속 가져가고 싶은 습관/사고방식은?
- **Problem**: 어려웠거나 부족했던 점은?
- **Try**: 다음에 시도해보고 싶은 것은? (실무 적용 계획)
