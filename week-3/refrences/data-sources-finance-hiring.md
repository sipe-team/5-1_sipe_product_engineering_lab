# 3주차 보조 데이터 가이드: 재무정보와 고용인원 숫자

3주차에서 공개 데이터를 활용해 프로덕트의 성공/실패를 추론할 때, 기업의 재무정보나 고용인원 데이터도 참고할 수 있습니다.

하지만, 이 데이터들은 보통 **핵심 증거** 라기보다 **보조 근거** 로 쓰는 것이 적절할 것으로 보입니다!

## 1. 재무정보는 언제 도움이 될까?

예: 매출, 영업이익, 당기순이익, 거래액, 구독 매출 비중 등

이런 정보를 보면 아래와 같은 질문에 답하는 데 도움이 됩니다.

- 이 회사가 전반적으로 성장 중인가?
- 수익성 압박이 큰가?
- 최근 어떤 사업 모델이 더 중요해지고 있는가?
- 왜 특정 기능이나 상품을 밀고 있는가?

예를 들어,

- 매출은 늘지만 이익이 악화된다면
  - 공격적인 확장 중이거나, 아직 효율이 낮을 수 있습니다.
- 이익률이 개선된다면
  - 가격 정책, 운영 효율화, 고마진 상품 확대 가능성을 의심해볼 수 있습니다.
- 특정 사업 부문 매출이 커진다면
  - 해당 제품군이나 고객군이 더 중요해졌다고 추론할 수 있습니다.

## 2. 고용인원 데이터는 언제 도움이 될까?

예: 월별 입사/퇴사 인원, 직군별 채용 증가, 조직 확장 여부

이런 정보는 회사가 **어디에 투자하고 있는지**, 혹은 **무엇을 축소하고 있는지**를 읽는 데 유용합니다.

- AI/데이터 직군 채용 증가
  - 추천, 자동화, 모델 고도화에 힘을 싣고 있을 가능성
- 세일즈/CS 채용 증가
  - 고객 확대, 엔터프라이즈 영업, 운영 강화 신호일 가능성
- 개발 채용 축소 또는 퇴사 증가
  - 우선순위 조정, 조직 불안정, 비용 절감 압박 가능성
- 특정 팀 중심의 채용 공고 증가
  - 앞으로 밀고 싶은 제품 영역이 무엇인지 추론 가능

## 3. 왜 보조 근거로 봐야 할까?

재무정보와 고용인원 데이터는 유효한 신호지만, 아래 한계가 있습니다.

- 회사 전체 단위 숫자인 경우가 많다
- 내가 분석하는 **특정 기능**과 직접 연결되지 않을 수 있다
- 외부 변수 영향을 많이 받는다
  - 경기, 투자 환경, 회계 처리, 구조조정, 계절성 등
- 시차가 크다
  - 재무 수치는 보통 제품 변화보다 늦게 반영된다

즉,

> 재무/고용 데이터는 "무슨 일이 벌어지고 있는가"를 보여주는 신호일 수는 있지만,
> "이 기능이 성공했다"를 직접 증명하는 증거는 아닐 수 있습니다.

## 4. 실전에서는 이렇게 쓰면 좋다

추천 우선순위는 아래와 같습니다.

### 1차 근거

- 사용자 리뷰
- 커뮤니티 반응
- 공개 지표
- 제품 기능 변화 이력
- 채용 공고
- 기술 블로그
- 뉴스/인터뷰/VOC

### 2차 근거

- 재무정보
- 고용인원 변화
- 투자/조직 개편 관련 기사

### 예시 문장

- "앱스토어 리뷰와 커뮤니티 반응을 보면 이 기능에 대한 불만이 반복적으로 보인다."
- "동시에 최근 분기 수익성 악화와 CS/운영 직군 채용 증가는, 회사가 사용자 만족보다는 운영 안정화와 비용 통제에 더 집중하고 있을 가능성을 시사한다."
- "다만 이는 회사 전체 단위 신호이므로, 특정 기능의 실패를 직접 증명한다고 보기는 어렵다."

## 5. 3주차에서 권장하는 사용 방식

- 재무정보/고용 데이터는 **있으면 활용한다**
- 하지만 이것만으로 결론 내리지 않는다
- 반드시 다른 사용자/제품 신호와 연결해서 해석한다
- 문서에는 아래처럼 쓰면 충분하다

```md
## 보조 근거
- 최근 채용 공고에서 AI/데이터 직군 비중이 높아졌다.
- 영업이익은 악화되었지만 매출은 성장 중이다.

## 해석
- 회사는 아직 수익 최적화보다 성장과 기능 고도화에 더 무게를 두고 있을 가능성이 있다.
- 다만 이는 회사 전체 관점의 신호이며, 내가 분석한 특정 기능의 성공을 직접 증명하는 근거는 아니다.
```

## 결론

- 재무 정보를 보는 것은, 프로덕트 분석에서 데이터를 분석하는 것에 **도움이 됩니다**
- 하지만 **이 숫자만으로 특정 기능의 성공/실패를 단정하면 안 됩니다**
- 사용자 반응, 기능 변화, 채용 공고, 리뷰, 커뮤니티 반응 같은 데이터와 **함께 해석** 해야 합니다
