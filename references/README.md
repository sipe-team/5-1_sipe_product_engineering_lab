# 📚 참고 자료

미션 진행 중 참고할 수 있는 자료들을 모아둔 공간입니다.

## Product Engineer란?

### 정의
> **프로덕트 엔지니어**는 서비스 성장을 최우선으로 두고, 프로덕트적인 관점에서 사고하는 엔지니어를 말합니다.
> — 당근 블로그

> 전통적 소프트웨어 엔지니어는 **Output**(코드)으로 측정되지만, 프로덕트 엔지니어는 **Outcome**(비즈니스 임팩트)으로 측정됩니다.

### 핵심 차이

| 구분 | 전통적 소프트웨어 엔지니어 | 프로덕트 엔지니어 |
|------|------------------------|----------------|
| 핵심 질문 | "어떻게 만들지?" (How) | "왜 만들지? 뭘 만들지?" (Why, What) |
| 측정 기준 | 코드 품질, 기술적 완성도 | 사용자 지표, 비즈니스 임팩트 |
| 입력 | 명세서, Jira 티켓 | 사용자 문제, 시장 데이터 |
| 의사결정 | PM이 결정한 것을 실행 | 스스로 무엇을 만들지 판단 |

### 주요 인용

> "최고의 개발 도구가 직접 코딩에서 AI 가이딩으로 전환될 것이다. **프로덕트 스킬에 더 집중하고** 최고의 도구를 계속 사용해야 한다."
> — John Carmack

> "프론트엔드 개발자라고 정의하는 순간, 서비스보다 기술이 그 직군의 정의에 먼저 오게 된다. 반면 프로덕트 엔지니어라고 하면, 가장 기대하는 게 **'서비스를 성장시키는 것'**이 된다."
> — 이용욱 (보이저엑스)

> "프로덕트의 생존과 성공보다 더 큰 포트폴리오는 없어"
> — 당근 알바팀
