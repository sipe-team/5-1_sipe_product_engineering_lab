# 📅 4주차: 개선 제안 — "내가 이 제품의 Product Engineer라면"

## 이번 주 목표
- PRD(Product Requirements Document) 형태로 개선 제안서 작성
- 서로의 PRD를 리뷰하며 토론

## PRD 작성 가이드

### 1. 문제 정의 (Why)
- 어떤 사용자가 어떤 상황에서 어떤 고통을 겪고 있는가?
- 이 문제를 방치하면 어떤 비즈니스 영향이 있는가?
- 근거 데이터/지표 (2~3주차 분석에서 가져오기)

### 2. 목표와 성공 지표 (What, KPI)
- 이 개선이 성공하면 어떤 상태가 되는가?
- 핵심 KPI와 목표치

### 3. 사용자 경험 변화 (Before → After)
- 현재 사용자 경험 vs 개선 후 사용자 경험

### 4. 구현 접근법
- 내가 이걸 만든다면 어떤 기술적 접근을 할 것인가 (러프한 수준 OK)

### 5. 우선순위 판단
- 왜 이걸 다른 것보다 먼저 해야 하는가?
- ICE Score: Impact(1~10) × Confidence(1~10) × Ease(1~10)

### 6. 리스크와 대안
- 예상 리스크와 실패 시 대응 방안

## 산출물

- PR로 제출 → 서로 코멘트
